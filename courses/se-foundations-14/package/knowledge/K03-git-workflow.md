# Git Workflow and Commit Quality

Git is not only backup. It is a record of project decisions.

Good commits are small, meaningful, and readable. A weak commit message says "update" or "fix stuff." A stronger one says "Add assignment status calculation" or "Fix submission scan for uploaded files."

A simple workflow:
1. Plan a change.
2. Make a small edit.
3. Test it.
4. Commit with a meaningful message.
5. Push when stable.
