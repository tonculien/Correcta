# Computer Science 2: Data Structures, Recursion, and Object-Oriented Design

## Course Description

This 21-day course continues from introductory programming into the core ideas of Computer Science 2. Students practice recursion, object-oriented design, linked structures, stacks, queues, trees, traversal, and basic algorithm analysis. The course is practical and assignment-driven: each concept is connected to code students must design, test, explain, and improve.

The course uses Python so students can focus on data structure behavior, design decisions, and correctness instead of excessive language boilerplate.

## Duration

21 days.

## Learning Objectives

By the end of the course, students will be able to:

- Write recursive functions with clear base cases and correct decomposition.
- Design small object-oriented modules with encapsulated state and testable behavior.
- Implement linked lists, stacks, and queues using node-based structures.
- Implement and explain tree traversal and search algorithms.
- Analyze time and space complexity using Big-O notation.
- Build a small data-structure-based application with tests and design documentation.

## Daily Structure

- Days 1-3: Recursion, call stack reasoning, and problem decomposition.
- Days 4-7: Object-oriented design, classes, methods, encapsulation, and testing.
- Days 8-11: Linked lists, stacks, queues, and edge-case handling.
- Days 12-15: Trees, traversal, search, and recursive data structures.
- Days 16-18: Algorithm analysis, complexity, and implementation tradeoffs.
- Days 19-21: Final project completion, testing, and explanation.

## Assignment List

- A01 Recursive Problem Solving: appears day 1, due day 4.
- A02 Object-Oriented Module Design: appears day 4, due day 8.
- A03 Linked List, Stack, and Queue Implementation: appears day 8, due day 12.
- A04 Tree Traversal and Search: appears day 12, due day 16.
- A05 Algorithm Analysis Report: appears day 16, due day 19.
- FINAL Data Structure Application: appears day 17, due day 21.

## Grading Overview

Grades are calculated using weighted average scoring. Each assignment is graded using its own 100-point rubric. The final project has the highest weight because it integrates course concepts into a complete working application.

Assignment weights:

- A01 Recursive Problem Solving: 1
- A02 Object-Oriented Module Design: 1
- A03 Linked List, Stack, and Queue Implementation: 1.25
- A04 Tree Traversal and Search: 1
- A05 Algorithm Analysis Report: 0.75
- FINAL Data Structure Application: 2

## Expected Learner Behavior

Students are expected to submit original work, read feedback carefully, revise when appropriate, and explain their reasoning in plain language. Students should test edge cases, document assumptions, and avoid submitting code they cannot explain.

AI tools may be used for study support, but submitted code and explanations must reflect the student's own understanding. Students should be able to describe how their code works, why their data structure choices are appropriate, and what tradeoffs exist.

## Final Outcome

Students will produce a small Python application that uses one or more data structures, includes meaningful tests, and contains a written explanation of design choices, complexity, and limitations.
